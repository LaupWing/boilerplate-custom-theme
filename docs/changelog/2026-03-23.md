# 2026-03-23

## Namespace rename: BP/Boilerplate -> SNEL

Complete rename of all namespace prefixes across all PHP, JS, JSON, CSS, and MD files:

- PHP function prefix: `bp_` -> `snel_` (e.g. `bp_get_lang` -> `snel_get_lang`, `bp__()` -> `snel__()`)
- PHP constants: `BP_OPENAI_API_KEY` -> `SNEL_OPENAI_API_KEY`
- JS global: `bpTranslate` -> `snelTranslate` (PHP `wp_localize_script` + all JS references)
- AJAX action: `bp_translate` -> `snel_translate` (PHP `add_action` + JS `FormData`)
- Block name prefix: `boilerplate/` -> `snel/` (block.json, seed data)
- Text domain: `'boilerplate'` -> `'snel'` (all `__()` and `_e()` calls)
- CSS/JS handles: `bp-` -> `snel-` (enqueue handles, `registerPlugin`, HTML element IDs)
- Theme slug: `Text Domain: snel` in style.css
- `@package` tags: `Boilerplate` -> `Snel` in all PHP docblocks
- WordPress option keys: `bp_theme_translations` -> `snel_theme_translations`, `bp_openai_*` -> `snel_openai_*`
- Admin page slugs: `bp-translations` -> `snel-translations`, `bp-seo` -> `snel-seo`, etc.
- HTML element IDs: `bp-lang-*` -> `snel-lang-*`, `bp-seo-root` -> `snel-seo-root`, etc.
- SOP.md updated with new prefix naming conventions

## New: translateBlockAttributes helper

Added `translateBlockAttributes()` to `src/blocks/components/lang-helpers.js` — translates block attributes by key with fresh state reads to avoid stale props when translating multiple languages sequentially.

## Updated: content-section edit.js

Updated `bpTranslate` -> `snelTranslate` references in the content-section block editor.
